# Project Blueprint Engine — Claude Code Context

This repository implements a **Specification-First** methodology for software development.

## What This Is

A structured framework that transforms a software idea into an implementation-ready specification pack before any code is written.

## Available Commands

| Command | Description |
|---|---|
| `/blueprint` | Start or continue the blueprint process (auto-detects current stage) |
| `/blueprint-jump` | Jump to a specific stage: `/blueprint-jump 5` |
| `/blueprint-status` | Show current progress and generated documents |
| `/spec-audit` | Run the Spec Auditor against all documents in `Specs/` |

## How It Works

1. Run `/blueprint` → it reads `Specs/.state.md` to determine the current stage
2. If no state exists → starts at Stage 1
3. After each stage → generates the corresponding document in `Specs/` and updates `.state.md`
4. After all 18 stages → run `/spec-audit` to validate consistency

## State File

The current stage is tracked in `Specs/.state.md`.  
Do not edit this file manually unless you need to reset or jump stages.

## Agent Files

- `BLUEPRINT_BUILDER_AGENT.md` — Full blueprint process instructions (18 stages)
- `SPEC_AUDITOR_AGENT.md` — Spec validation rules and output format

## Critical Rules

- **Never write implementation code** during the blueprint phase
- **Never skip stages** — each stage depends on the previous one
- **Always save generated documents** to the `Specs/` folder
- **Always update** `Specs/.state.md` after completing a stage
- The `Specs/` folder is the single source of truth for the project specification
