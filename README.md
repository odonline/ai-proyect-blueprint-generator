# 🛠️ AI-Assisted Project Specification Engine

> **"Idea → Architecture → Domain → Specs → Code"** — *Because random code leads to architectural chaos.*

## Overview

A structured framework that converts software ideas into **implementation-ready specifications** using a **Specification-First** methodology.

Unlike traditional AI coding approaches that jump from prompt to code, this engine enforces a disciplined pipeline where architecture and domain are fully defined before any implementation begins.

---

## The Pipeline

```
User Idea
    ↓
Blueprint Builder Agent  (/blueprint)
    ↓
18-Stage Guided Q&A
    ↓
Spec Documents (Specs/)
    ↓
Spec Auditor Agent  (/spec-audit)
    ↓
Validated Spec Pack
    ↓
Implementation AI (your dev agent)
    ↓
Working Codebase
```

## Getting Started

### Option A — Standalone (new project)

```bash
git clone https://github.com/your-org/blueprint-engine my-project
cd my-project
```

### Option B — Add to existing project (git submodule)

```bash
git submodule add https://github.com/your-org/blueprint-engine .blueprint
```

Then copy the `.claude/` folder to your project root:

```bash
cp -r .blueprint/.claude ./
cp .blueprint/BLUEPRINT_BUILDER_AGENT.md ./
cp .blueprint/SPEC_AUDITOR_AGENT.md ./
cp .blueprint/CLAUDE.md ./
mkdir -p Specs
```

## Usage with Claude Code

Open the project folder in Claude Code, then use these slash commands:

### `/blueprint`
Start or continue the blueprint process.

Claude auto-detects the current stage from `Specs/.state.md` and resumes from where you left off.

```
> /blueprint
```

First run starts Stage 1 — Problem Definition.
Subsequent runs continue from the last completed stage.

### `/blueprint-status`
View current progress across all 18 stages.

```
> /blueprint-status
```

### `/blueprint-jump <N>`
Jump to a specific stage (use with caution — skipping stages breaks consistency).

```
> /blueprint-jump 8
```

### `/spec-audit`
Run the full spec auditor against all generated documents.

```
> /spec-audit
```

Generates `Specs/SPEC_AUDIT_REPORT.md` with all detected issues.

## Why Specification-First?

| Traditional Flow | Blueprint Flow |
|---|---|
| Prompt → AI writes code | Idea → 18-stage spec → Validated pack → Code |
| Architecture invented on the fly | Architecture defined before any code |
| Contradictions discovered in production | Contradictions caught at spec stage |
| Impossible to onboard new AI agents | Any agent can implement from the spec |

---

## After the Spec Pack is Complete

Hand the `Specs/` folder + `ai_control_layer/` to your implementation agent.

The spec pack is the contract. The implementation agent must not deviate from it.
