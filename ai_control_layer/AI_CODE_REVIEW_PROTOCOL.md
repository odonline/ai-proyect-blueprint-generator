# AI Code Review Protocol

## Purpose

This protocol defines the review process for any code generated by AI systems.

The objective is to ensure that AI-generated code does not violate domain rules, architecture constraints, or system invariants.

---

# Step 1 — Domain Validation

Verify that the AI did not modify:

- Domain entities
- Entity relationships
- Cardinality rules
- Domain invariants

Reference document:

DOMAIN_SOURCE_OF_TRUTH.md

If any change is detected → reject the code.

---

# Step 2 — State Machine Validation

Verify that the AI did not modify:

- State definitions
- State transitions
- Terminal states
- Transition rules

Reference document:

STATE_MACHINE_SPEC.md

If any change is detected → reject the code.

---

# Step 3 — Database Validation

Verify that the AI did not modify:

- Table structure
- Primary keys
- Foreign keys
- Unique constraints
- ENUM definitions

Reference document:

DB_SCHEMA.sql

If any change is detected → reject the code.

---

# Step 4 — Financial Logic Validation

Verify that the AI did not modify:

- PaymentEvent types
- Payment flows
- Refund logic
- Currency handling

Reference document:

BACKEND_IMPLEMENTATION_CONTRACT.md

---

# Step 5 — Architecture Validation

Verify that the AI respected the architecture layers:

controllers → services → repositories → database

Forbidden:

controllers → repositories  
controllers → database

---

# Step 6 — Test Coverage

Verify that AI-generated code includes:

- Unit tests
- State transition tests
- Constraint validation tests

Minimum coverage:

70%

---

# Final Decision

If any violation is detected:

1. Reject the generated code
2. Request re-generation with corrections
3. Document the violation
